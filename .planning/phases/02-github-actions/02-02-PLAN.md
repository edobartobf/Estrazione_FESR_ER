<objective>
Create the GitHub Actions workflow file that automates weekly FESR scraping.

Purpose: The workflow triggers automatically every Monday at 06:00 UTC and computes
the previous week's date range (Mon–Sun). It can also be triggered manually with
custom keyword and dates. The scraper is driven entirely via FESR_* env vars;
no credentials or values are hardcoded.

Output: .github/workflows/fesr_scraper.yml ready to commit and push.
</objective>

<context>
From Plan 02-01: scraper.py reads FESR_KEYWORD, FESR_DATA_DA, FESR_DATA_A, FESR_ANNI,
FESR_OUTPUT from os.environ via applysecrets() called at the start of main().

Date computation logic (Python):
  from datetime import date, timedelta
  today = date.today()
  last_monday = today - timedelta(days=today.weekday() + 7)
  last_sunday = last_monday + timedelta(days=6)
  data_da = last_monday.strftime("%d/%m/%Y")
  data_a  = last_sunday.strftime("%d/%m/%Y")

weekday() returns 0=Monday … 6=Sunday.
today.weekday() + 7 always gives the offset to the previous Monday
(7 days back if today is Monday, more otherwise).
</context>
